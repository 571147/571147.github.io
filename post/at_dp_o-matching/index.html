AT_dp_o Matching

2023-04-25 3 min read # AtCoder # 题解

题目

  • 有 nnn 个男人和 nnn 个女人。
  • 已知每个男人和每个女人是否可以配对。
  • 要求配成 nnn 对，每对一男一女。
  • 求方案数  mod  109+7\bmod~10^9+7mod 109+7。
  • 1≤n≤211\le n\le 211≤n≤21.

状态压缩 DP 做法

我们设一个集合 SSS，t=∣S∣t=|S|t=∣S∣ 且里面只包含若干个女人的编号，定义一个数组 dpSdp_{S}dpS​，表示前 ttt 个男人与 SSS 中的女人匹配的方案数，可以枚举第 ttt 个男人和哪个女人匹配，很明显，状态转移方程为：

dpS=∑dpS−i(i∈S and at,i=1)dp_S=\sum dp_{S-i}(i\in S~\text{and}~a_{t,i}=1) dpS​=∑dpS−i​(i∈S and at,i​=1)

当然，表示一个集合 SSS 太复杂了，我们可以把它转化为一个 nnn 位二进制数，从右往左的第 iii 位如果为 111，表示集合中有编号为 iii 的女人，否则表示没有。那么状态转移方程就变为：

dpS=∑dpS−2i−1(⌊S2i−1⌋ mod 2=1 and at,i=1)dp_S=\sum dp_{S-{2^{i-1}}}(\left\lfloor\frac{S}{2^{i-1}}\right\rfloor\bmod 2=1~\text{and}~a_{t,i}=1) dpS​=∑dpS−2i−1​(⌊2i−1S​⌋mod2=1 and at,i​=1)

枚举所有集合，设集合为 SSS、t=∣S∣t=|S|t=∣S∣，枚举第 ttt 个男人和哪个女人匹配，复杂度为 O(n2n)O(n2^n)O(n2n)，nnn 只有 212121，不会 TLE。

代码中要用到一些 C++ 常用的位运算符，详见 https://blog.csdn.net/digitalkee/article/details/117634604。

代码实现

#include<bits/stdc++.h>
using namespace std;
const int mod=1e9+7;
int n,a[22][22],dp[1<<21];
int main(){
	scanf("%d",&n);
	for(int i=1;i<=n;i++)
		for(int j=1;j<=n;j++) scanf("%d",&a[i][j]);
	dp[0]=1;//集合里没有女人，肯定也没有男人，方案数为 1
	for(int S=1;S<1<<n;S++){
		int S2=S,t=0;
		while(S2>0) t+=S2&1?1:0,S2>>=1;//求有多少个女人，有多少个女人就有多少个男人
		for(int i=1;i<=n;i++)
			if(a[t][i]==1&&S>>i-1&1)//判断第 t 个男人是否能和第 i 个女人匹配且集合中有这个女人
				dp[S]=(dp[S]+dp[S-(1<<i-1)])%mod;
	}
	printf("%d",dp[(1<<n)-1]);
	return 0;
}
