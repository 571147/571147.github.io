P9198 「GMOI R2-T1」轴对称

2023-04-25 4 min read # 洛谷 # 题解

题目

  • 有一个 nnn 行 mmm 列的图片，该图片的像素为 n×mn\times mn×m，每个格子有一个初始 RGB 值 (0,0,0)(0,0,0)(0,0,0)。
  • 有 qqq 个操作，每个操作是将第 iii 行第 jjj 列的格子的 RGB 值的第 ttt 个数增加 ccc，需要对 256256256 取模。
  • 每个操作后要判断这个图片是否左右对称。
  • 1≤n,m≤1001\le n,m\le 1001≤n,m≤100，1≤q≤5001\le q\le 5001≤q≤500，1≤i≤n1\le i\le n1≤i≤n，1≤j≤m1\le j\le m1≤j≤m，t∈{1,2,3}t\in \{1,2,3\}t∈{1,2,3}，1≤c≤1091\le c\le 10^91≤c≤109。

做法

这道题数据范围较小，可以使用 O(qnm)O(qnm)O(qnm) 的暴力法，每次修改后把图片遍历一次判断是否左右对称即可，这里就不给出代码了。

还有一种做法非常高效，可以应对 1≤n,m≤1091\le n,m\le 10^91≤n,m≤109，1≤q≤5×1051\le q\le 5\times 10^51≤q≤5×105 的数据范围，前置知识是 STL map （单次操作复杂度为 log⁡n\log nlogn 级别，1≤q≤5×1051\le q\le 5\times 10^51≤q≤5×105 是没问题的）。

设 color(i,j),tcolor_{(i,j),t}color(i,j),t​ 表示第 iii 行第 jjj 列的格子的 RGB 值的第 ttt 个数，numnumnum 为第 iii 行第 jjj 列与第 iii 行第 m−j+1m-j+1m−j+1 列的格子的 RGB 值不同的 (i,j)(i,j)(i,j) 的组数。对于每次操作，要比较第 iii 行第 jjj 列与第 iii 行第 m−j+1m-j+1m−j+1 列的格子的 RGB 值：

  • 修改前相同：若修改后不同，num←num+1num\gets num+1num←num+1，否则不变。
  • 修改前不同：若修改后相同，num←num−1num\gets num-1num←num−1，否则不变。

之后只需要判断 num=0num=0num=0 是否成立即可。

代码实现

#include<bits/stdc++.h>
using namespace std;
const int mod=256;
map<pair<int,int>,map<int,int> >color;
bool check(int i,int j,int m){
	bool f1=color[{i,j}][1]==color[{i,m-j+1}][1];
	bool f2=color[{i,j}][2]==color[{i,m-j+1}][2];
	bool f3=color[{i,j}][3]==color[{i,m-j+1}][3];
	return f1&&f2&&f3;
}
int main(){
	int n,m,q,num=0;scanf("%d%d%d",&n,&m,&q);
	while(q--){
		int i,j,t,c;scanf("%d%d%d%d",&i,&j,&t,&c);
		if(check(i,j,m)){
			color[{i,j}][t]=(color[{i,j}][t]+c)%mod;
			if(!check(i,j,m)) num++;
		}
		else{
			color[{i,j}][t]=(color[{i,j}][t]+c)%mod;
			if(check(i,j,m)) num--;
		}
		if(num==0) printf("Yes\n");
		else printf("No\n");
	}
	return 0;
}
